STEP 5. FOCUS ON "convert_number" function (this function is inside 11-file)

	Step 5.1 (convert_number function)

		- The "convert_number" function (11-file) is called when the number is not found directly in "numbers.dict".
		- This function convert_number(char *n, t_dict *dict, int size) (i.e. "number string", "structs", "struct size")-consecutively.
		- The function mentioned converts a numeric string into words using grouped processing.
		- The function starts by calculating the length of the input number using "ft_strlen".
		- The return value of the "ft_strlen" will be used to compute how many 3-digit groups are needed using arithmetic grouping logic.
		- Once the number of groups is identified, it will be stored in g variable. then proceed to Step 5.2. 
		
	Step 5.2 (get_group function 11-file)

		- The g value will then be use to extract the 3-digit block of the "number argument string".
		- The g value will then be passed to the static "get_group" function (11-file).
		- This function takes "get_group(char *n, char *buf, int g)" (i.e. "number argument string", "buffer size", g value)-consecutively.
		- It calculates the correct position inside the string.
		- It copies up to three digits into the buffer.
		- If the position is out of range, it fills missing digits with '0'.
		- It ensures every group is always normalized to 3 digits.

	Step 5.3 (main loop inside convert_number in 11_convert_large.c)

		- The function enters a loop controlled by the group counter.
		- For each iteration, it calls get_group to extract the current block.
		- It then sends the group to convert_hundreds (10_convert_hundreds.c).
		- After that, it calls handle_tens (also in 10_convert_hundreds.c).
		- These two functions convert the 3-digit group into words using dictionary lookup.

	Step 5.4 (scale handling using print_scale and get_scale_key in 11_convert_large.c)

		- After processing each group, the function checks if a scale is needed.
		- It uses print_scale (static function inside convert_number file).
		- The scale word is retrieved using dict_get (07_dictionary_lookup.c).
		- The key for the scale is generated using get_scale_key (static function in same file).
		- This determines whether to print thousand, million, billion, etc.
		- The scale is only printed if the group contains a non-zero value.
		- This prevents printing unnecessary scale words for empty groups.

	Step 5.5 (dictionary lookup inside dict_get in 07_dictionary_lookup.c)

		- The scale word is validated through dict_get function.
		- This function compares generated scale keys with dictionary keys.
		- When a match is found, it returns the corresponding textual value.
		- This returned value is passed into print_word (12_output.c).

	Step 5.6 (output control using print_word in 12_output.c)

		- The final word output is handled by print_word.
		- This function prints each character of the word using write.
		- It ensures clean output formatting for each converted token.

	Step 5.7 (spacing control inside convert_number in 11_convert_large.c)

		- The variable "first" is used to control spacing.
		- It ensures that spaces are only printed between valid words.
		- The first printed word does not have a leading space.
		- All following words are preceded by a space.

	Step 5.8 (loop completion inside convert_number in 11_convert_large.c)

		- The loop continues until all groups are processed from highest to lowest.
		- Each group is independently converted and printed.
		- After finishing all groups, a newline is printed using write.

	Step 5.9 (function termination in convert_number in 11_convert_large.c)

		- The function returns success after completing the full conversion.
		- Control returns back to main.c after execution.
